# Teensy 4.1 ボタン配線・実装手順 (2x4 マトリクス)

Game Boy の 8 ボタン（十字キー4 + A/B/Start/Select）を、GB 実機と同じ
**2 列 × 4 行のマトリクス**で Teensy 4.1 に接続する手順。GPIO 6 本で 8 ボタンを
読み取れる。実装は [`teensy/src/input.rs`](../teensy/src/input.rs) の `GpioInput`。

---

## 1. 必要な部品

| 部品 | 数量 | 備考 |
|---|---|---|
| タクトスイッチ（モーメンタリ） | 8 | 押している間だけ ON |
| ダイオード | 8 | **ショットキー推奨**（例: BAT43）。1N4148 でも可（後述） |
| ジャンパー線 | 適量 | |
| プルアップ抵抗 | **不要** | Teensy 内部 22k プルアップを使用 |

> ダイオードは**ゴースト（同時押し時の誤検出）防止に必須**。例えば「右 + A + B」を
> 同時に押すと、ダイオードが無い場合は「左」が誤検出される（マトリクスの回り込み）。

---

## 2. ピン割り当て

| 役割 | 信号 | Teensy ピン | GPIO | 向き |
|---|---|---|---|---|
| 列選択（方向キー） | SEL_DIR | 28 | GPIO3[18] | 出力 |
| 列選択（アクション） | SEL_ACT | 29 | GPIO4[31] | 出力 |
| 行入力 0 | IN0 | 30 | GPIO3[23] | 入力（内部プルアップ） |
| 行入力 1 | IN1 | 31 | GPIO3[22] | 入力（内部プルアップ） |
| 行入力 2 | IN2 | 32 | GPIO2[12] | 入力（内部プルアップ） |
| 行入力 3 | IN3 | 36 | GPIO2[18] | 入力（内部プルアップ） |

ピンと GPIO ポートの対応は `teensy4-pins` の型で固定されており、`input.rs` で
誤ったポートを渡すとコンパイルエラーになる（実装上の安全網）。

---

## 3. マトリクス表（どのボタンがどの交点か）

各ボタンは「列線（SEL）」と「行線（IN）」の交点に置く。

|              | IN0 (pin30) | IN1 (pin31) | IN2 (pin32) | IN3 (pin36) |
|--------------|-------------|-------------|-------------|-------------|
| **SEL_DIR (pin28)** | 右 Right    | 左 Left     | 上 Up       | 下 Down     |
| **SEL_ACT (pin29)** | A           | B           | Select      | Start       |

---

## 4. 1 ボタンの配線（ダイオードの向き）

ボタンは「SEL 列線」と「IN 行線」の間に、ダイオードと直列に入れる。
**ダイオードのアノードを IN 側、カソードを SEL 側**に向ける（電流は IN→SEL の向き）。

```
 IN行線 ──┤>├── [ SW ] ── SEL列線
          ▲アノード   ▲カソード(向きは ─┤>├─ の通り IN→SEL)

   (押下していない時)  IN行線は内部プルアップで HIGH
   (押下した時)        その列が LOW なら IN行線が LOW に引かれる → 押下と判定
```

向きの理由:
- 走査中、選択した列だけ LOW・他の列は HIGH に駆動する。
- 選択列(LOW)のボタンを押すと、IN 行 → ダイオード → 列(LOW) に電流が流れ、
  IN が LOW に落ちる（= active-low、押下）。
- 非選択列(HIGH)側はダイオードが逆バイアスで遮断され、行に影響しない（ゴースト防止）。

> **ダイオードの電圧降下**: 1N4148 等は約 0.7V 降下するため、押下時の IN は約 0.7V。
> 3.3V ロジックの VIL（約 1.0V）は下回るので読めるが、マージンを確保したいので
> **ショットキー（降下約 0.3V）を推奨**。`input.rs` では入力にヒステリシス（Schmitt）も
> 有効化してチャタリング耐性を上げている。

## 5. 全体配線図

```
Teensy 4.1                            ボタンマトリクス
──────────                            ──────────────────────────────
pin28 SEL_DIR ──┬──────────┬──────────┬──────────┐  (列: 方向キー)
                │          │          │          │
              [右]       [左]       [上]       [下]
                │          │          │          │
                ┤>├        ┤>├        ┤>├        ┤>├   (各ボタンにダイオード)
                │          │          │          │
pin29 SEL_ACT ──┼───┐      ┼───┐      ┼───┐      ┼───┐ (列: アクション)
                │   │      │   │      │   │      │   │
                │  [A]     │  [B]     │ [Sel]    │ [Start]
                │   │      │   │      │   │      │   │
                │   ┤>├    │   ┤>├    │   ┤>├    │   ┤>├
                │   │      │   │      │   │      │   │
pin30 IN0 ──────┘   │      │   │      │   │      │   │
pin31 IN1 ──────────┘      │   │      │   │      │   │
pin32 IN2 ─────────────────┘   │      │   │      │   │
pin36 IN3 ─────────────────────┘      ... (各 IN 行に 2 ボタンずつ接続)
```

> 上図は概念図。実際は **IN0 行に「右」と「A」**、IN1 行に「左」と「B」…という具合に、
> 各 IN 行へ方向キー列とアクション列の 2 ボタンがダイオード経由でぶら下がる。
> グラウンド配線は不要（列線を LOW に駆動して代用するため）。

## 6. ソフトウェアの動作（実装済み）

[`teensy/src/input.rs`](../teensy/src/input.rs) の `GpioInput`:

1. **初期化 (`new`)**
   - IN0–IN3 に `iomuxc::configure` で**内部 22k プルアップ + ヒステリシス**を設定。
   - SEL_DIR/SEL_ACT を出力にし、待機時は両方 HIGH（非選択）。
2. **走査 (`poll`)** — 1 フレームに 1 回呼ばれる
   - `SEL_DIR=LOW, SEL_ACT=HIGH` → `SETTLE_DELAY`(≈5µs) 待ち → IN0–IN3 を読み
     右/左/上/下 に格納（`is_set()==false` = 押下）。
   - `SEL_ACT=LOW, SEL_DIR=HIGH` → 同様に A/B/Select/Start を読む。
   - 最後に両列 HIGH に戻す。

`main.rs` で `board::t41` から `gpio2/gpio3/gpio4` を取り出し、確定ピンで
`GpioInput::new(...)` を構築している。ソフト側の追加作業は不要。

## 7. 動作確認

| チェック | 期待 |
|---|---|
| 何も押さない | 全ボタン非押下（ゲームが入力なしで進む） |
| 十字キー各方向 | キャラ/カーソルが対応方向に動く |
| A / B | 決定 / キャンセル等が反応 |
| Start / Select | ポーズ / メニューが反応 |
| 右 + A + B 同時押し | 「左」が誤入力されない（= ダイオードが効いている） |

**うまくいかない場合:**
- 全ボタンが効かない → SEL_DIR(28)/SEL_ACT(29) の配線、または IN 行の接続を確認。
- 特定の 1 ボタンだけ効かない → そのボタン/ダイオードのはんだ・向きを確認。
- 押すと別のボタンが反応 → ダイオードの向き（アノード=IN 側）を確認。
- 押下が不安定 → ダイオードの電圧降下が大きい可能性。ショットキーに交換。
- 走査が速すぎて読めない（基本起きないが） → `SETTLE_DELAY` を増やす。
