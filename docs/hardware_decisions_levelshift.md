# ハードウェア設計決定事項 — レベルシフタ・ピンアサイン

作成日: 2026-06-27  
対象: `teensy/src/cartridge.rs` / KiCad PCB 設計

> **改訂 (2026-06-27 設計レビュー)**: `teensy/src/cartridge.rs` の確定ピン表と突き合わせ、以下を修正。
> 1. データバスを単方向 245 × 2（U4/U5）→ 双方向 245 × 1（U4, DIR=/RD 直結）に統合。IC 総数 5 → 4。
> 2. ピン表の A14 (pin 6) を `GPIO4` → `GPIO2[10]`（`GPIO_B0_10`）に訂正。
> 3. §7「A0–A9 は連続ビット」を訂正（同一 GPIO1 レジスタだがビットは非連続）。

---

## 1. レベルシフタ IC の選定

### 採用: 74AHCT245

GB カートリッジは 5V 系、Teensy 4.1 は 3.3V 系（5V トレラントピンなし）。
電圧変換に **74AHCT245** を採用する。

**採用理由:**
- AHCT 系は入力閾値が TTL 互換（H 閾値 ≈ 2.0V）のため 3.3V 入力で確実に H 判定できる
- プッシュプル出力のため駆動能力が高く、波形が崩れにくい
- GB カートリッジの最大動作周波数（CLK 4.194MHz）に対して十分なマージンがある
- TXS0108E と比較してプルアップ依存の立ち上がり遅延がなく波形品質が優れる

**不採用: TXS0108E**  
プルアップ抵抗経由の出力構造のため立ち上がりが遅く、容量性負荷に弱い。
GB バスには過剰なリスクがある。

## 2. 必要 IC 数と配置

74AHCT245 は 8ch/IC。信号線の方向別に以下の構成とする。

| # | 対象信号 | ch 数 | 方向 | 備考 |
|---|----------|-------|------|------|
| U1 | A0–A7 | 8 | Teensy → Cart | 単方向 |
| U2 | A8–A14, /RESET | 8 | Teensy → Cart | 単方向。A15 は不使用 |
| U3 | /RD, /WR, /CS | 3（+空き5ch） | Teensy → Cart | 単方向。CLK は未接続のため非搭載（§6 参照）。空き ch は将来拡張用 |
| U4 | D0–D7 | 8 | **双方向** | DIR=/RD で方向切替（§3 参照） |

**合計: 4 個**

> 旧版は D0–D7 に単方向 245 を 2 個（書き込み用 U4・読み込み用 U5）割り当てていたが、
> 74AHCT245 は DIR ピンを持つ双方向トランシーバなので 1 個で双方向バスを賄える。
> 2 個構成は冗長で、両方を常時イネーブル（/OE=GND）にすると同一ネットを 2 IC が
> 並列駆動してしまうため不可（§3 参照）。

---

## 3. データバス方向制御

### DIR 信号の配線

データバス（D0–D7）は双方向のため、双方向トランシーバ 74AHCT245（U4）1 個で構成し、
DIR ピンで方向を切り替える。**DIR 専用 GPIO は使用せず、/RD 信号（pin 33）を DIR に直結する。**
A 側を Teensy、B 側を Cart に割り当てると、74AHCT245 の DIR は「H で A→B」のため、
/RD をそのまま接続するだけで正しい向きになる（**インバータ不要**）。

```
Teensy pin 33 (/RD) ──┬──→ カートリッジ /RD（U3 経由）
                       │
                       └──→ U4 DIR
```

| /RD の状態 | 意味 | DIR | データバスの向き |
|------------|------|-----|------------------|
| H（非読み込み） | 書き込みまたはアイドル | H (A→B) | Teensy → Cart |
| L（読み込み中） | カートリッジ読み取り | L (B→A) | Cart → Teensy |

> 旧版は単方向 245 を 2 個使い、真理値表では「U4 有効／U5 無効」の排他制御を意図していたが、
> 実際の配線（/OE 全 GND＋DIR 制御）では両 IC が常時イネーブルのままで排他にならず、
> 同一ネットを 2 IC が並列駆動してしまう。1 個の双方向 245 に統合すればこの矛盾は解消する。

### /OE ピンの扱い

U1–U4 すべての `/OE` ピンは **GND に固定**する（常時イネーブル）。GPIO ピンを消費しない。

> **補足（リセット時の安全策）**: Teensy リセット中は GPIO が Hi-Z 入力になり、/OE=GND の 245 は
> 入力フロートのまま出力を駆動する。制御線 /RD・/WR・/CS の Teensy 側にプルアップ（非アサート=H に固定）を
> 入れておくと、リセット中にカートリッジへ誤ったアサートが伝わるのを防げる。/RD のプルアップは
> U4 DIR を Teensy→Cart 側（カート Hi-Z なので無害）に倒す効果も兼ねる。

## 4. /RESET ピンの制御

### 変更前（旧設計）

カートリッジの `/RESET` を 3.3V に固定接続していた。

### 変更後（今回決定）

**pin 37 を /RESET 制御用 GPIO として割り当てる。**

```
Teensy pin 37 → 74AHCT245 (U2 の空き ch 経由) → Cart /RESET
```

**変更理由:**  
MBC1/MBC3/MBC5 付きカートリッジはリセット時にバンクレジスタが不定状態になる。
電源投入後に一定時間 LOW → HIGH のリセットパルスを与えることで初期化を保証する。
3.3V 固定では RomOnly カートリッジしか安定動作しない。

### ソフトウェア対応（`cartridge.rs`）

```rust
// GpioCart::new() 内で実行
reset_pin.set_low();
cortex_m::asm::delay(600_000); // 1ms @ 600MHz
reset_pin.set_high();
```

## 5. 5V 電源供給

カートリッジ VCC (5V) は **Teensy 4.1 の VIN ピン**（USB 5V 直結）から供給する。

### デカップリングコンデンサ

カートリッジコネクタの VCC–GND 間に以下を**並列**に配置する。

| 部品 | 値 | 役割 | 配置 |
|------|----|------|------|
| 電解コンデンサ | 100µF | 低周波の電流需要（突入電流バッファ） | コネクタ近傍 |
| MLCC | 100nF | 高周波スイッチングノイズの除去 | コネクタ直近 |

各 74AHCT245 の VCC(pin20)–GND(pin10) 間にも **100nF MLCC を 1 個ずつ**配置する（計 4 個）。
IC ピンから 2mm 以内に実装すること。

## 7. ソフトウェア残タスク（cartridge.rs）

### A10–A14 の scatter 実装

A0–A9 は全て同一の GPIO1 レジスタに載るため 1 回の DR 書き込みで出力できる
（ただしビットは**非連続**: 2,3,12,13,16,17,26,27,28,29 なので scatter 処理が必要）。
A10–A13 は GPIO4、A14 は GPIO2 に分散しているため個別にセットする必要がある。

### /RESET 制御の追加

```rust
// GpioCart 構造体に reset_pin フィールドを追加
pub struct GpioCart {
    // ... 既存フィールド
    reset_pin: GpioPin<Output>,  // pin 37
}

impl GpioCart {
    pub fn new(..., reset_pin: ...) -> Self {
        let mut cart = Self { ..., reset_pin };
        cart.reset();
        cart
    }

    pub fn reset(&mut self) {
        self.reset_pin.set_low();
        cortex_m::asm::delay(600_000); // 1ms @ 600MHz
        self.reset_pin.set_high();
        cortex_m::asm::delay(600_000); // 安定待ち
    }
}
```
